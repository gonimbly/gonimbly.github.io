Setup

The two step process for deploying Engager's API and UI. Deploy your own version of Engager using the Heroku buttons below:

Engager API

Engager UI

Deploying API to Heroku using the Command Line

  1. Clone the API repository:

    git clone https://github.com/gonimbly/engager-api

  2. Create the Heroku app:

    heroku create

  3. Install Postgres:

    heroku addons:create heroku-postgresql:hobby-dev

  4. Install Redis:

    heroku addons:create heroku-redis:hobby-dev

  5. Deploy the Engager API:

    git push heroku master

  6. Initialize the database:

    heroku run bin/init-db

Deploying UI to Heroku using the Command Line

  1. Clone the UI repository:

    git clone https://github.com/gonimbly/engager-ui

  2. Create the Heroku app:

    heroku create

  3. Deploy the UI:

    git push heroku master

Running Locally

Two services need to be setup and started in order for the application to run locally. Also for the sake of easy setup, we'll be using the externally hosted heroku-postgres and heroku-redis databases to store information.

  1. Clone the API repository:

    git clone https://github.com/gonimbly/engager-api

  2. Install the dependencies:

    npm i

  3. Retrieve database environment variables Heroku (must run "heroku addons:create .." first):

    heroku config

  4. Set local environment variables (note ssl=true):

    export DATABASE_URL="postgres://{connection_str}?ssl=true"
    export REDIS_URL="redis://{connection_str}"

  5. Initialize the database:

    ./bin/init-db

  6. Start the API:

    npm run start

  7. In a new terminal tab, navigate up a directory and clone the UI repository:

    git clone https://github.com/gonimbly/engager-ui

  8. Install the dependencies:

    npm i

  9. Change the endpoint in index.html (note to use "http" if local, "https" if production):

    API_URL = "http://localhost:9001";
