Setup

The two step process for deploying Engager's API and UI. Deploy your own version of Engager using the Heroku buttons below. Please note this UI is only meant to be used in Mobile Browsers.

Heroku Buttons

Engager API

On successful deployment, the herokuapp root will show a "Thank you" message.

Engager UI

Use url from API as config variable API_URL w/o the ending slash, ex: "https://engager-api123.herokuapp.com"

Using Heroku Connect for Answers (optional)

  1. Install the unmanaged package to your desired salesforce org first.

  2. In Heroku UI of your new app, provision herokuconnect addon, via quick search addons menu.

  3. Click the Heroku Connect addon, after creation to configure the service to your Salesforce instance.

  4. Click next till you authorize the app and are at the Mappings configuration screen.

  5. Create New Mapping

  6. Select the managed package SObject "engager_answer__c" as the desired object.

  7. Check the "Database->Salesforce" option and select the following fields: value__c, customer_name__c, questions_text__c.

  8. Save.

  9. Test by answering a question and see the answer in Salesforce "Engager" tab a few moments later

  10. More info on Heroku Connect here.

Deploying API to Heroku using the Command Line

  1. Clone the API repository:

    git clone https://github.com/gonimbly/engager-api

  2. Create the Heroku app:

    heroku create

  3. Install Postgres:

    heroku addons:create heroku-postgresql:hobby-dev

  4. Install Redis:

    heroku addons:create heroku-redis:hobby-dev

  5. Deploy the Engager API:

    git push heroku master

  6. Initialize the database:

    heroku run bin/init-db

Deploying UI to Heroku using the Command Line

  1. Clone the UI repository:

    git clone https://github.com/gonimbly/engager-ui

  2. Create the Heroku app:

    heroku create

  3. Deploy the UI:

    git push heroku master

  4. Set the API config variable:

    heroku config:set API_URL='https://sample-api-url.herokuapp.com'

Running Locally

Two services need to be setup and started in order for the application to run locally. Also for the sake of easy setup, we'll be using the externally hosted heroku-postgres and heroku-redis databases to store information.

  1. Clone the API repository:

    git clone https://github.com/gonimbly/engager-api

  2. Install the dependencies:

    npm i

  3. Retrieve database environment variables Heroku (must run "heroku addons:create .." first):

    heroku config

  4. Set local environment variables (note ssl=true):

    export DATABASE_URL="postgres://{connection_str}?ssl=true"
    export REDIS_URL="redis://{connection_str}"

  5. Initialize the database:

    ./bin/init-db

  6. Start the API:

    npm run start

  7. In a new terminal tab, navigate up a directory and clone the UI repository:

    git clone https://github.com/gonimbly/engager-ui

  8. Install the dependencies:

    npm i

  9. Set the API config variable:

    export API_URL='https://localhost:9000'

  10. Start the UI:

    npm start
